# AGENT.md

## 项目定位

这是一个面向学习和实践的 Node.js AI Agent 项目。

核心原则：

- 先理解原理，再堆功能
- 先做最小闭环，再做抽象扩展
- 保持模块边界清晰，方便后续重构
- 每个阶段都能运行、验证、复盘

## 当前版本目标

当前阶段目标是实现一个从零开始、逐步演进的 Agent 系统，能力按以下顺序增长：

1. 对话
2. 上下文记忆
3. 工具调用
4. 多步任务执行
5. 可观测与可测试
6. 高级 Agent 能力

## Agent 核心组成

### 1. Model Layer

职责：

- 封装大模型 API 调用
- 统一模型输入输出格式
- 处理重试、超时和错误

建议文件：

- `src/llm/client.js`
- `src/llm/prompts.js`

### 2. Memory Layer

职责：

- 管理对话历史
- 控制上下文长度
- 支持短期记忆与持久化存储

建议文件：

- `src/agent/memory.js`

### 3. Tool Layer

职责：

- 注册工具
- 校验参数
- 执行工具
- 返回工具结果

建议文件：

- `src/tools/registry.js`
- `src/tools/builtins/*.js`

### 4. Agent Core

职责：

- 组合模型、记忆和工具
- 管理一次任务的执行过程
- 决定何时继续、何时结束

建议文件：

- `src/agent/core.js`
- `src/agent/loop.js`

### 5. Interface Layer

职责：

- CLI 交互
- 参数输入
- 输出运行结果和日志

建议文件：

- `src/index.js`

## 推荐开发原则

### 保持小步迭代

每次只增加一个可验证能力，例如：

- 今天先实现模型调用
- 下一步再加对话历史
- 再下一步加一个时间查询工具

### 保持模块单一职责

避免把这些逻辑混在一个文件里：

- 用户输入处理
- API 调用
- 工具注册
- 会话存储
- 执行循环

### 先跑通，再优化

先追求：

- 能运行
- 能观察
- 能解释

再考虑：

- 抽象优雅
- 高级模式
- 性能优化

## 里程碑定义

### M1: Chat Agent

能力：

- 用户输入问题
- Agent 调用模型
- 返回文本结果

验收：

- 支持基本对话
- 支持系统提示词

### M2: Memory Agent

能力：

- 多轮对话
- 保存会话上下文

验收：

- Agent 能引用前文信息

### M3: Tool Agent

能力：

- 能调用本地工具
- 能把工具结果再交给模型处理

验收：

- 至少 2 个工具可用
- 有完整 tool call 流程

### M4: Task Agent

能力：

- 执行多步骤任务
- 有基础规划和循环

验收：

- 能完成一个多阶段任务
- 具备中间过程输出

### M5: Reliable Agent

能力：

- 日志
- 配置
- 测试
- 错误处理

验收：

- 关键流程稳定
- 可以调试和排查问题

## 技术决策建议

### Node.js 版本

建议使用较新的 LTS 版本，优先保证：

- 原生 `fetch`
- 良好的 ESM 支持
- 更稳定的异步能力

### 模块规范

建议尽早决定使用：

- ESM

原因：

- 更贴近现代 Node.js 项目
- 与前端和现代工具链更统一

### 日志策略

最初阶段可以使用简单 `console`，但要区分：

- 用户可见输出
- 调试日志
- 错误日志

### 测试策略

建议至少覆盖：

- Memory 裁剪逻辑
- Tool 注册与调用
- Agent loop 停止条件

## 开发纪律

- 新功能优先补充到对应模块，不要把逻辑堆回入口文件
- 每完成一个阶段，都更新 README 的进度
- 每增加一个工具，都记录输入、输出和失败场景
- 每次重构，都保证最小闭环还能运行
