# 学习计划

这个文档用于记录项目的学习路线、阶段规划和阶段性待办。

## 学习目标

1. 熟悉 Node.js 项目结构、模块拆分、异步流程和错误处理。
2. 理解 AI Agent 的核心组成：模型、记忆、规划、工具、执行循环。
3. 从零实现一个可观察、可调试、可扩展的 Agent。
4. 在实现过程中沉淀文档、实验记录和阶段复盘。

## 产出目标

1. 一个基于 Node.js 的命令行 AI Agent。
2. 一套清晰的模块化工程结构。
3. 一份从入门到进阶的开发路线与里程碑。
4. 一套便于后续扩展 Web UI、插件系统和多 Agent 能力的基础架构。

## 开发愿景

项目分成三层目标：

- 第一层：做出最小可用 Agent，能接收输入并调用模型返回结果。
- 第二层：加入记忆、工具调用、日志和配置管理，让 Agent 具备基础执行能力。
- 第三层：加入规划、任务拆解、长期记忆、评估和多 Agent 协作等进阶能力。

## 阶段规划

### Phase 0: 项目初始化

目标：完成工程骨架和开发约定。

已完成内容：

- 初始化 Node.js 项目
- 配置 `package.json`
- 建立目录结构
- 接入环境变量与用户配置管理
- 初始化 README、AGENT 文档
- 配置 ESLint / Prettier

完成状态：

- 已完成

### Phase 1: 最小 Agent 闭环

目标：实现一个最简单的对话式 Agent。

计划内容：

- 接收用户输入
- 调用大模型 API
- 返回模型回复
- 支持基础系统提示词
- 支持单轮/连续对话模式

当前进度：

- CLI 交互已完成
- LLM 接入层已完成
- 正在持续优化结构和可维护性

### Phase 2: 上下文与记忆

目标：让 Agent 能“记住”对话上下文。

计划内容：

- 设计消息结构（system / user / assistant / tool）
- 保存对话历史
- 支持历史裁剪与上下文窗口控制
- 支持本地文件持久化会话

### Phase 3: 工具调用能力

目标：让 Agent 不只是聊天，还能执行动作。

计划内容：

- 定义 Tool 接口
- 实现本地工具注册机制
- 接入简单内置工具
- 打通 tool call 执行循环

### Phase 4: Agent Loop 与任务执行

目标：从“对话机器人”升级为“任务执行器”。

计划内容：

- 实现 `plan -> act -> observe -> reflect` 基础循环
- 支持最大迭代次数
- 支持中间步骤日志输出
- 支持任务完成判断

### Phase 5: 可观测性与工程化

目标：让项目更像一个真正可维护的工程。

计划内容：

- 日志分级
- 配置中心
- 错误分类
- 单元测试
- Mock 模型层
- 增加开发脚本

### Phase 6: 进阶能力

目标：探索更真实的 Agent 架构能力。

计划内容：

- 短期记忆 / 长期记忆
- RAG 检索增强
- 简单规划器
- 任务拆解
- 多工具编排
- 多 Agent 协作实验
