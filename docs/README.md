# Docs

这个目录用于存放项目过程文档，而不是项目的主说明。

根目录 [README.md](../README.md) 负责回答：

- 这个项目是什么
- 现在能做什么
- 怎么启动和使用

`docs/` 目录负责回答：

- 这个项目准备怎么学、怎么推进
- 某个阶段做了什么
- 某个设计为什么这样定
- 调试和实验过程中得到了什么结论

## 当前文档

- [learning-plan.md](learning-plan.md)
  项目的学习目标、阶段规划、阶段待办和验收标准。
- [next-actions.md](next-actions.md)
  当前最值得优先处理的下一步建议。

## 推荐放置内容

后续可以继续在这里补充：

- 阶段复盘
- 设计笔记
- 调试记录
- 协议兼容说明
- 工具系统设计
- memory / loop / planner 实验记录

## 文档约定

- 根目录 `README.md` 只保留项目说明，不堆阶段计划。
- 学习路线、任务拆分、阶段复盘统一放在 `docs/`。
- 当某个主题已经超出 README 的“快速理解”范围时，优先单独建文档。
- 文档文件名尽量直接表达主题，例如：
  - `learning-plan.md`
  - `next-actions.md`
  - `tool-system-notes.md`
  - `memory-design.md`
  - `phase-2-review.md`

## 当前状态

目前项目处于 Phase 2，`docs/` 主要承载学习计划、近期建议和阶段进度。
